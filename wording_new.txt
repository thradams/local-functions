6.4.3.2 Predefined identifiers

Semantics

   The identifier __func__ shall be implicitly declared by the translator
   as  if,  immediately  following  the  opening  brace  of each function
   definition, the declaration

   static const char __func__[] = "function-name";

   appeared,  where  function-name is the name of the lexically-enclosing
   function.50)

   For function literals, the function-name is unspecified.

6.5.3.7 Function literal

Syntax

    function-literal:
       (attribute-specifier-sequence opt declaration-specifiers
           abstract-declarator) function-body

    function-body:
       compound-statement

Constraints

   1 The abstract-declarator shall declare a function type.

   2  The function-specifier, if any, in the function specifiers shall be
   _Noreturn.

   3  The storage-class specifier in the function specifiers shall not be
   used.

Semantics

   4 The function literal is a function designator.

   5 Distinct function literals are not required to have unique addresses

   6  Semantics from function definitions that apply to function literals
   are described in the section on function definitions.

6.9.2 Function definitions

Syntax
    function-definition:
        attribute-specifier-sequenceopt declaration-specifiers declarator function-body

    function-body:
         compound-statement

  
   Functions  can  be defined either as external declarations or as block
   items  within  other  functions.  Function literals also share certain
   constraints and semantics with function definitions.

Constraints

   2  The identifier declared in a function definition (which is the name
   of  the  function)  shall  have  a  function type, as specified by the
   declarator portion of the function definition.

   3  The  return  type  of a function or function literal shall be  void 
   or a complete object type other than array type.

   4  The  storage-class specifier, if any, in the declaration specifiers
   of  a  external defined function shall be either extern or static. 
   For block defined  functions, it  must be static. 

   5  If  the parameter list consists of a single parameter of type void,
   the parameter declarator shall not include an identifier. This also, 
   applies for function literals.

   6 Variable length array types of unspecified size shall not be used as
   part of a parameter declaration in a function definition.This also, 
   applies for function literals.

   Semantics

   7  The  optional attribute specifier sequence in a function definition
   appertains to the function or function literal.

   8  The  declarator  in a function definition specifies the name of the
   function being defined and the types (and optionally the names) of all
   the parameters; the declarator also serves as a function prototype for
   later  calls  to  the  same function in the same translation unit. The
   type of each parameter is adjusted as described in 6.7.7.4.

   9  If a function or function literal that accepts a variable number of
   arguments  is defined without a parameter type list that ends with the
   ellipsis notation, the behavior is undefined.

   10  The  parameter  type list, the attribute specifier sequence of the
   declarator  that  follows  the  parameter  type list, and the compound
   statement of the function body form a single block.192) Each parameter
   has  automatic  storage  duration;  its  identifier, if any,193) is an
   lvalue.194) The layout of the storage for parameters is unspecified.
   This also applies for function literals.

   11  On  entry  to  the function, the size expressions of each variably
   modified  parameter  and  typeof  operators  used  in  declarations of
   parameters  are evaluated and the value of each argument expression is
   converted  to  the  type  of  the  corresponding  parameter  as  if by
   assignment.  (Array  expressions and function designators as arguments
   were  converted  to  pointers before the call.). This also applies for
   function literals.

   12  For  localy  defined  functions  and  non extern function literals
   expressions  using  variably modified as return type, the VM types are
   evaluated at the point of function definition.

   13  After all parameters have been assigned, the compound statement of
   the function body is executed.

   14  Unless  otherwise specified, if the } that terminates the function
   body  is  reached,  and  the value of the function call is used by the
   caller, the behavior is undefined.

   15  Labels  declared  in an enclosing function are not shared with any
   block defined function or function literal.

   16  Variables, tags, and enumerators declared in an enclosing function
   are  accessible  within  the  body  of  a  locally defined function or
   function literal.

   17  An object with automatic storage duration declared in an enclosing
   function  shall  not be evaluated within a block defined  function  or
   function  literal.  An  exception is made for constexpr objects, which
   may  be  evaluated as if they were named constants and not lvalues; in
   particular, their address shall not be taken.
